멀티 윈도우 지원 계획

Tauri native window를 중심에 두고, 단일 창에도 항상 탭을 표시하며, macOS native tab 병합과 분리까지 확장 가능한 구조로 정리한다.

Goal

작업 목표는 CLI가 아니라 앱 내부의 멀티 윈도우 기반이다. `ma <filename>`은 후속 작업으로 분리한다.

Implementation Timeline

구현은 native window 경로를 먼저 고정한다

Phase 01

정적 window 제거

`app.windows`를 비우고 `RunEvent::Ready`에서 main 안내/예제 창을 생성한다.

Phase 02

결정적 label

Markdown 경로를 canonicalize하고 hash label로 중복 창을 판별한다.

Phase 03

query 기반 로딩

새 창 URL에 `path`를 넣고 프론트가 mount 시 문서를 읽는다.

Phase 04

하이브리드 탭

`tabbing_identifier`, `addTabbedWindow_ordered`, `toggleTabBar` 보정을 붙인다.

Key Interfaces

핵심 코드는 창 생성 경계에 모인다

src-tauri/src/inbound/window_commands.rs 
#[tauri::command]
fn request_open_document_window(
    app: AppHandle,
    path: String,
) -> Result<(), String> {
    let canonical = validate_markdown_path(&path)?;
    let label = label_for_document(&canonical);

    if focus_if_open(&app, &label) {
        emit_window_highlight(&app, &label);
        return Ok(());
    }

    create_document_window(&app, &label, &canonical)
}
Frontend Contract

프론트는 window query를 읽어 자기 문서를 로드한다

Document window

`?path=`가 있는 창

mount 시 `readMarkdownDocument(path)`를 호출하고 annotation, selection, dialog 상태를 초기화한다.

Welcome window

`?path=`가 없는 창

기존 예제 문서 또는 시작 화면을 보여준다. 파일 열기는 백엔드에 새 native window 생성을 요청한다.

features/open-document/windowQuery.ts 
export function getDocumentPathFromWindowQuery() {
  const params = new URLSearchParams(window.location.search);
  return params.get("path");
}
Decisions

전략은 native tab bar 하나로 통일한다

Required

단일 창 tab bar

window 생성과 포커스 시 native tab bar 표시를 보정한다.

Required

native tab group

모든 문서 창에 `tabbing_identifier("markdown-annotator")`를 적용한다.

Decide

파일 선택 후 현재 native tab group에 새 문서 창을 붙이는 방향을 우선한다.

Risks & Mitigations

리스크는 초기 로딩과 macOS 전용 동작에 집중된다

MED

이벤트 race condition

새 창 생성 직후 이벤트를 emit하면 프론트 구독 전에 놓칠 수 있다.

초기 문서는 이벤트가 아니라 `index.html?path=...` query로 전달한다.

MED

경로 비교 실패

상대 경로, symlink, 대소문자 차이로 같은 파일 판정이 흔들릴 수 있다.

모든 중복 판정은 canonical path에서 만든 deterministic label로 처리한다.

LOW

단일 native tab bar 한계

Tauri/wry 환경에 따라 `toggleTabBar`가 기대처럼 동작하지 않을 수 있다.

실패 시 hidden helper tab 또는 별도 AppKit 보조 전략을 검토한다.
